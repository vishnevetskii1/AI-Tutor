# Карта Проекта

Этот файл объясняет, за что отвечает каждая часть проекта.

## Корень проекта

- `main.py` - точка входа, запуск бота, регистрация команд и callback-обработчиков
- `.env` - шаблон runtime-настроек с заглушками
- `.env.example` - эталонный шаблон тех же настроек
- `Dockerfile` - сборка образа бота
- `docker-compose.yml` - локальный стек: bot, qdrant, ollama
- `pyproject.toml` - метаданные проекта и зависимости Python
- `uv.lock` - зафиксированные версии зависимостей
- `README.md` - установка и запуск
- `START_HERE.md` - короткий стартовый файл для телефона
- `PROJECT_MAP.md` - полная карта кода
- `ingest.py` - CLI-точка входа для индексации PDF
- `audit_rag.py` - CLI-проверка качества PDF и RAG
- `extract_tasks.py` - CLI-помощник для извлечения практических задач

## Папка `bot/`

- `config.py` - читает переменные окружения и собирает конфиг
- `graph_runner.py` - общий доступ к графу, локи по пользователям, чтение и обновление состояния
- `access_control.py` - ограничения доступа и проверки admin-only сценариев
- `course_sessions.py` - хранение активной дисциплины пользователя
- `disciplines.py` - названия, описания и вспомогательные функции по дисциплинам
- `levels.py` - подписи уровней пользователя
- `progress.py` - XP, ачивки и сводка прогресса
- `formatting.py` - форматирование текста плана, прогресса и служебных ответов
- `telegram_text.py` - разбиение длинного текста под лимиты Telegram
- `system_checks.py` - runtime-проверки и диагностика
- `remote_ops.py` - удалённые сервисные операции и их описание
- `topic_web_search.py` - внешний поиск по теме для lesson chat
- `lesson_export.py` - экспорт урока в Markdown и HTML

## Папка `bot/handlers/`

- `commands.py` - команды: `/start`, `/progress`, `/reset` и другие
- `messages.py` - обработка обычных текстовых сообщений пользователя
- `callbacks.py` - обработка inline-кнопок
- `ui.py` - общие клавиатуры и безопасная отправка длинных ответов

## Папка `bot/agents/`

- `graph.py` - сборка LangGraph-графа и маршрутизация узлов
- `state.py` - схема общего состояния, с которым работают все узлы
- `orchestrator.py` - решает, какой шаг должен выполниться следующим
- `profile_agent.py` - онбординг и определение уровня пользователя
- `planner_agent.py` - генерация учебного плана
- `content_agent.py` - генерация учебных блоков
- `qa_agent.py` - ответы на уточняющие вопросы по текущей теме
- `quiz_agent.py` - генерация вопросов квиза
- `evaluator.py` - проверка ответов и обновление прогресса
- `llm.py` - единый слой вызова моделей
- `generation_utils.py` - общие вспомогательные функции для парсинга и очистки вывода
- `input_validation.py` - валидация входных данных
- `response_cleanup.py` - финальная очистка сгенерированного текста перед отправкой

## Папка `bot/rag/`

- `chunker.py` - режет PDF-текст на чанки
- `ingest.py` - загружает чанки в Qdrant
- `pdf_audit.py` - проверяет, пригоден ли PDF для RAG
- `search.py` - поиск в Qdrant и резервный поиск по PDF

## Папка `bot/practice/`

- `extractor.py` - собирает датасеты с практическими задачами

## Папка `bot/maintenance/`

- `normalize_progress_plans.py` - сервисный скрипт нормализации сохранённых планов

## Папка `data/`

- `disciplines/` - сюда кладутся PDF по предметам, по папке на дисциплину
- `practice/` - сюда могут складываться JSON с практическими задачами

В этой чистой копии обе папки оставлены пустыми специально.

## Папка `docs/`

- `architecture.html` - визуальная схема архитектуры
- `user-guide.html` - пользовательская инструкция

## Основной рабочий поток

1. Пользователь отправляет `/start`
2. `bot/handlers/commands.py` запускает стартовый сценарий
3. `bot/graph_runner.py` поднимает state пользователя и вызывает граф
4. `bot/agents/orchestrator.py` решает, какой агент нужен дальше
5. Один из агентов строит план, урок, квиз или ответ
6. `bot/handlers/callbacks.py` обрабатывает нажатия кнопок и ведёт пользователя дальше
